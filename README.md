# Mimir FrameWorks 大模型对话游戏主持框架

## 1. **项目背景与目标**

本项目旨在开发一个基于大语言模型（LLM）的对话游戏主持框架，支持类似DND（龙与地下城）、狼人杀等对话类游戏的自动化主持。框架需要具有高度的可扩展性和适应性，以便在不同游戏环境下稳定运行，而不需要对核心大模型进行频繁修改。

## 2. **模块设计**

整个系统划分为三个主要模块：Adapter模块、游戏逻辑模块和LLM核心模块。各模块独立运作，分工明确，但通过精细的接口设计和数据交互紧密联系，确保系统的灵活性和可扩展性。

### 2.1 **Adapter模块**

**功能**：负责与外部平台的交互，包括QQ机器人、Web界面、其他聊天软件等，处理所有的输入和输出。

**组件**：

- **平台接口（Platform Interfaces）**：为每个外部平台创建独立的接口类（例如，QQBotInterface、WebInterface、ChatGroupInterface），负责特定平台的API调用、消息格式处理和响应。
- **消息路由器（Message Router）**：一个中心路由器，根据上下文或用户输入将消息分发到相应的游戏逻辑模块或LLM核心模块。
- **API管理器（API Manager）**：管理与外部API的通信，处理请求和响应。

**框架化设计**：此模块设计为高度模块化，支持外部平台接口的快速添加或移除，而核心逻辑无需更改。

**与其他模块的联系**：Adapter模块接收到外部输入后，会通过消息路由器将信息传递给游戏逻辑模块。游戏逻辑模块处理后产生的输出，再通过消息路由器传递到LLM核心模块，生成最终的响应。最终响应由Adapter模块发送回外部平台。

### 2.2 **游戏逻辑模块**

**功能**：负责管理特定游戏的规则、交互/UI和机制。

**组件**：

- **游戏引擎（Game Engines）**：为每个游戏创建独立的引擎或类（例如，DNDEngine、WerewolfEngine），包含特定游戏的规则、交互和决策逻辑。
- **状态管理器（State Manager）**：管理游戏状态，跟踪玩家操作，确保交互的一致性。
- **交互处理器（Interaction Handler）**：处理玩家输入，触发游戏事件，并与LLM核心模块进行交互以生成动态响应。

**框架化设计**：该模块设计为灵活的，可根据需要轻松切换或更新不同的游戏引擎，而不影响其他系统模块的正常运行。

**与其他模块的联系**：游戏逻辑模块不仅从Adapter模块接收玩家的输入信息，还可以根据游戏需求向LLM核心模块提出对基础设定库或知识库的修改请求。例如，某个游戏场景可能需要更新知识库中的某些内容，游戏逻辑模块会通过接口调用LLM核心模块的知识库管理器，进行动态调整。LLM核心模块处理后，返回的内容由游戏逻辑模块使用，并决定下一步的游戏操作。

### 2.3 **LLM核心模块**

**功能**：作为系统的核心“大脑”，负责生成响应、管理知识库，并适应不同的游戏场景，类似于定制一个 Agent。

**组件**：

- **基础设定库（Base Setting Library）**：一组预定义的指令和参数，引导LLM在游戏主持人角色中生成合适的输出（例如，“你是一个DND主持人，你需要在获得xxx信息后，输出xxx”）。
- **知识库管理器（Knowledge Base Manager）**：管理知识库或设定库的动态注入，基于用户输入或预设的游戏场景进行调整，而无需修改LLM核心。
- **微调模块（Fine-Tuning Module）**：（可选）一个机制，用于在必要时通过特定数据集对LLM进行微调，增强其作为游戏主持人的能力。
- **响应生成器（Response Generator）**：基于游戏逻辑模块和Adapter模块的输入，生成上下文相关的响应。

**框架化设计**：LLM核心模块设计为稳定的中央组件，尽量避免频繁变动。通过知识库管理器进行的调整，可以适应不同的游戏设定和输出需求，而无需修改底层模型。

**与其他模块的联系**：LLM核心模块主要通过响应生成器接收游戏逻辑模块的输入，并基于基础设定库和知识库生成对应的游戏响应。同时，知识库管理器可以根据游戏逻辑模块的请求进行实时调整，确保LLM的输出符合当前游戏的需求。LLM核心模块生成的最终响应通过Adapter模块反馈给用户。

## 3. **系统集成与运行流程**

- **初始化**：系统启动时，加载游戏逻辑模块中的相应游戏引擎，加载Adapter模块中的相关平台接口，并将相应的知识库或设定库加载到LLM核心模块中。
- **运行时**：在游戏进行过程中，Adapter模块处理外部输入，游戏逻辑模块处理特定的游戏逻辑，并根据需要与LLM核心模块进行交互以生成动态响应。LLM核心模块生成的响应由Adapter模块传递回用户。
- **自定义**：如果需要引入新游戏或修改现有游戏，仅需更新或替换游戏逻辑模块。LLM核心模块和Adapter模块保持不变，确保系统的稳定性和一致性。
